## はじめに

まず謝罪します。筆者はこういうダメなタイトルを試したかっただけです（変な悪趣味）。どれくらいの人がクリックするか見てみたかった 😂

クリックしたあなたへ —— I got you !

元々 1 年達成時に書こうと思っていたのですが、その時は本当に忙しかった（はいはい、毎回そう言ってるのは知ってる）。なので本来「1 年の振り返り」のはずが、さらに 2 ヶ月以上（もうすぐ 3 ヶ月）引き伸ばされました。**筆者は正直者なので**、1 年 2 ヶ月は 1 年 2 ヶ月です。「1 年転職振り返り」なんて言いません。2 ヶ月でも結構なことができます。3 秒で何ができるかは、あなたが自分で知っていればいいんです。

もっと正確に言うと：

```javascript
const milliSeconds = new Date() - new Date('2021/11/15') // 筆者成為開發者第一天上班日
const year = milliSeconds / 1000 / 60 / 60 / 24 / 365
console.log(year)
// 1.176578354578894 年
```

筆者はできるだけ分かりやすい言葉でこの振り返りを書くつもりです。初心者、転職中の方、非専門家の方にも読んでもらえることを想定し、皆さんのお役に立てればと思います。もう一つ先に言っておくと、もし文中にあなたを傷つける内容があれば、立ち止まってこの道が本当に自分に合っているか考えてみてください。あるいは無視しても構いません。でも真実を言う人は必要で、嫌われても仕方ありません。

もし **軽くさらっと成功を描いた振り返り** のほうが好きなら、誰でも簡単にできそうな錯覚を起こすようなやつ、それが役立つならたくさん読めばいい！筆者も言った通り、私は正直者です。幻覚なんかで私を脅せません！

だからこそ、もう少しシリアスな振い返りを書こうと思ったのです。多くの場合、早めに諦めさせるほうが時間の節約になります。早めに損切りすれば、もっと重要なことに取り組けるし、高い受講料も浮いて、もっと好きなこと、あるいはプログラミングより得意なことに時間を使えます。プログラミングは転職の唯一の選択肢ではありません！

転職前のストーリーはこちら：[\[MTR-05\] 学生の振り返り：Benben](https://github.com/Lidemy/mentor-program-5th/issues/31)。ここでは転職後のストーリーに焦点を当てます。

はいはい、年収百万にする方法を知りたいですよね。でもまずは読み進めましょう。

## どうしてそんなことできたの？

- 3 ヶ月未満で転職？

筆者が最初に勤めたのは、あるデジタルデザイン会社 K です。当時「メタバース」ブームに乗っかっていて、将来性がありそうだと思って入社しました。入ってから気付きました、私は間違っていたと！マネージャー/PM は技術を全然理解しておらず、常態的な残業など。でも誰のせいでもありません。むしろ一つの教訓を得ました。メタバース、仮想通貨、NFT などのキーワードを見たら警戒すること。新しい言葉に乗ってトラフィックを稼いでいるだけかもしれません。会社のバックグラウンドをしっかり調べましょう。

同時に働きながら観察もしました。**試用期間は会社があなたを見ているだけでなく、あなたも会社を観察している期間です。**本当にダメならすぐに変えましょう。時間を無駄にしないこと。「1 年未満だと履歴書が悪く見える」なんて気にしなくていい。例えば、同じ会社に 5 年いた人の履歴書は良い履歴書でしょうか？それとも実力が足りずに飛べないから留まっているだけ？一緒に働かないと分かりません。1 年未満での転職なんて本当に大したことありません。あなたにはあと何回「1 年」をこうして試せると思いますか？

その後、結構いい小さな会社 R に移りました。K を離れたかったので 104 を開きました。年の瀬だったこともあり、面接のオファーはたくさん来ました。前回の面接経験があったので、明らかにヤバそうな会社は最初にフィルターしました。R 社の CTO といっぱい話しました。一気に 2 時間。その場でオファーをもらいました。オファーも良くて、迷うこともなく移りました。

- 半年で特休を使って面接に？

筆者自身、**半年くらいで自分の履歴書を整理する**タイプです。多くの大先生方もそう言っているので、改めて言うまでもないでしょう。定期的に機会を見ることで、自分に何が足りないか、自分の価値がどの水準にあるかも分かります。すべてあなたのプラスになります。やらない手はありません。

他人の目が気になる？やめましょう。離職後も連絡を取る友達がどれだけいますか？一人の視線に未来を遅らせる必要がありますか？でも「行き過ぎ」もダメです。入ってすぐに転職したがり、ずっと求人を見て、ずっと面接に行く、いわゆる「転職病」だと、自分を磨かず、長期的には単に「面接がうまいだけ」になります。筆者は 3〜6 ヶ月くらいの頻度をおすすめします。もちろん、あなたが既にトップ企業や上級職にいるなら別です。

- 小まとめ

実は筆者もかなり運が良かったです。R 社の CTO は本当にいいマネージャーでした。技術の深さも幅もあり、App を書きながら PM もやり、無茶な要求も無茶な会議もなく、退勤後は全く邪魔せず、グループチャットは誰もいないかのように静か。部下にも割と良くしてくれました。ただ彼もとても忙しかったですが。小さな会社ですからね。普通です。

私が「辞めるからといって仕事をしない」わけではありません。退職前日まで 8 時間しっかり働き、ドキュメントを書き、過去のコードを整理しました。在職約 1 年の間、遅刻も早退もありませんでした。全力で会社のためにコードを書き、タスクが終わっても「どこを最適化できるか」を考え続けました。この仕事に関しては愧じることはないと言えます。それで十分だと思います。これ以上続けていたら、もっと抜け出しづらくなっていたでしょう。

R 社を離れるのは遅かれ早かれのことでした。私の作業量はもう一人のフロントエンドのほぼ 2〜3 倍でした。もう一度言います、**筆者は正直者です**。もちろん CTO も本当に忙しくて、気付かなかったのかもしれません。退職を申し出た日の午後、CTO といろいろな考えを話しました。彼が「将来マネージャーになりたいか」と聞いてきて、私は即座に「はい！」と答えました（じゃなきゃなんで磨き続けてるの？マネージャーやシニアになりたいんです、希望ですが）。彼はただ「それならもう少し多くやるのはいいことだ。彼（CTO 自身）の作業量は人 5 人分に換算できるが、私はせいぜい 2 人分（10 人分は無理 QQ）」と簡単に言いました。私は彼の心の中で彼の要求に達したかどうかも、自分の選択が正しかったかも分かりません。でも天のみぞ知る、でしょうか。

その日の退勤後、內湖からスクーターで麥帥一橋を渡って帰る途中、夜空は澄んでいました。隣の 101 はまだ微かな青い光を放ち、ゆっくりと、点滅し、点滅していました。

## なぜ私にその資格が？

- ポートフォリオ

多くのフロントエンド開発者はポートフォリオを整理するのが嫌いです。私もです！

プラットフォームを変えるたびに、履歴書もポートフォリオも一からやり直しです。名前、生年月日、住所、そして一番面倒な自己紹介。考えるだけで疲れるので、私はこういうものを整理し続けるのが本当に嫌です。ソフトウェア開発の世界では古くから「Don't repeat yourself」と言われています。繰り返しは一度でいい。自分の情報や自己紹介を何度も書くのも、ある意味 repeat yourself です（笑）。

フロントエンドエンジニアとして、自分のウェブサイトを作るのは日常茶飯事でしょう。だから自分のポートフォリオをちゃんと整理し、リスト形式で作品を並べるくらい、難しいことじゃないですよね？でもこの時点で「すごく情熱がある」というフロントエンド開発者は姿を消します。口をそろえて「フロントエンドにすごく情熱がある」と言うのに、GitHub は空っぽ（プライベートプロジェクトや他のプラットフォームにある可能性は排除しませんが）。だから筆者みたいに正直者になりましょう。ないならないと言えばいい。情熱があるなら自然と表れます。無理に装う必要はありません。

履歴書とポートフォリオサイトを作った後、筆者はいくつか **オリジナル** や **共同** 作品もあり、真面目に語ればそれぞれ 1 時間くらいは話せます。皆さんにお願いします、**ポートフォリオは絶対にコピーしないでください**。そうすれば履歴書の返信率はぐっと上がります。筆者は秘訣を全部教えました。隠してなんかいませんよ XD。

- アルゴリズム

多くのフロントエンド開発者は問題をたくさん解くのが嫌いです。私もです！

まず、「ドリル」が嫌いなだけで、「解く」感覚は好きです。主に JavaScript と TypeScript を使います。これらはフロントエンドの主要言語でもあるので、言語とロジックの両方に慣れられます。今は言語間の差はどんどん小さくなっています（動的/静的、強型/弱型の違いはもちろんありますが）。他人ができることは JavaScript でもできます（待って、言語戦争するつもりはありません）。問題演習に使っても全然問題ありません。無理に Python や C++ に合わせる必要はありません（本当に入門するなら C++ をおすすめします）。

問題演習で鍛えるのは各プログラミング言語ではなく、論理的思考だと思います。低レイヤの実装は単なるディテールです。変数を `let`、`const`、`int`、`bool`、`char*` のどれで宣言するかなど、自分の慣れた言語を使えばいい。後から第二、第三の言語に挑戦すればいいのです。

2023/01/15 に [2023 Yourator デジタルキャリアEXPO](https://www.accupass.com/event/2211140558381956060137) に行きました。筆者はあちこちで「フロントエンドの面接で LeetCode は出ますか？」と聞きました。答えはほとんど **出ない、あるいは easy が 1 問程度** でした。もし演習が本当に嫌ならやらなくても OK ですが、その分、実務経験で実力を証明する必要があります。

- Vim

多くのフロントエンド開発者は Vim を使うのが嫌いです。私もです！

筆者が「嫌い」と言っているのは、Vim を学び始めた頃のこと。終了（`:q`）すら半日ハマり、移動は 1 文字ずつ（`hjkl`）、入力は `i` だけ、保存して終了は（`:wq`）だけ。でも基本、これだけで **すべてのタスク** を完了できます。ただ……時間がかかるかどうかだけ。

その後、いろんな大先生の神プレイを見て、一生懸命練習しました。筆者は本当に上記のような状態から Vim を使い始め、ゆっくりコードを書き始めました。最終的に Vim の中を飛び回り、各種スクリプトを使い、操作は行雲流水のように。古代の神器でフロントエンドを書くとかっこいいことこの上ない（X。

Vim を学ぶのはあまり役に立たないように見えます。例えば、Vim の `:q` は quit の意で、多くのソフトウェアがこの伝統に従っています。`git log` で `q` を入力するのも終了の意味です。またリモートサーバーに入るとデフォルトで Vim が使われます。この時、過去の自分が Vim を学んでいてくれてありがとうと思うでしょう！

Twitter やコミュニティでエディタ戦争が起きるたび、**real pro use vim** という言葉を見かけます。プロの開発者は Vim で開発するという意味です。現在、フロントエンドで Vim を使って開発している人はほとんどいないでしょう（待って、エディタ戦争するつもりはありません）。便利なツールも増え、Vim ができることは他でもできるようになっています。だから学ぶかどうかは人次第です。

要するに、**生産性を高められる**ものは何でも良いものです。Emacs や IntelliJ を試すのもいい選択です。ただこの二つは筆者はあまり触っていません。

- 小まとめ

気付きましたか？ **他人が嫌うことをやる** —— それがあなたの強みになります。もちろん学ぶ努力も必要です。優秀とは何か？それは **他人を平凡にする能力** です。筆者も同意します。「卷（巻き・内巻）」だと思うかもしれませんが、仕方がありません。世界はそう動いているのです。あなたのスキルが希少で需要があればあるほど、高級人材になります。あるいは考えてみてください。Junior と Senior の違いは何ですか？同じ年数なのに、Senior があなたではなく他人なのはなぜ？

待って！あなたもCSS切りを嫌いだって？それならいいですよ、バックエンドの SQL はどうですか？

エンジニアジョークにこんなのがあります：フロントエンドは CSS が切れない、バックエンドは SQL が書けない。

もちろん、これらを「包んで」くれるツールもたくさんあります。あまり適切ではない例ですが：Bootstrap が CSS 切りを代替し、ORM が SQL を代替します。でも Bootstrap を変更しようとしたら変更できず、簡単な CRUD の SQL が書けない時、あなたは気付くでしょう。本当にエンジニアが毎日コピペしてお茶を飲んでいるだけで給料をもらっているとでも？そんな夢のようなバブル、筆者が先に突いてあげましょう。もしそんな仕事が本当に存在するなら、すぐに私に教えてください、人募集してるか（えっ？）。

## やったこと

- **新技術** を追い続け、自分に欠けている **旧技術** を補う

ソフトウェア分野で唯一変わらないのは、常に変化していること。これも言い古されました。特にフロントエンドは、フレームワークやツールの変化が本当に激しい！ついていかないと淘汰されます。React はもう 18 です。jQuery だけで足りますか？Vite のビルドは稲妻のように速い、Vue 3 も魅力的、学んでみませんか？

新技術を追うのは開発者にとって日常茶飯事。では……旧技術は？**計算機概論、データ構造、アルゴリズム**。面白いことに、これらは万年変わらない旧技術なのに、学ぶ人は少ない。新技術と比べて CP 値が最も高いはずなのに、みんな新技術に夢中、あるいは新技術の優劣を議論しています。議論する時間があるなら自己研鑽に使いましょう。これらを身につければ、そもそも議論する必要もありません。

情報工学科の学生は 4 年かけて培った実力です。3〜6 ヶ月で追いつけるとでも？まだ時間があるなら、これらの必須技術知識をしっかり補いましょう。転職のオーラにはいつか尽きる日が来ます。未来の転職するあなたは、過去のあなたに感謝するでしょう（ややこしい）。視野を長期に持ちましょう。

> 筆者が他にどんな本を読んでいるか気になる場合は：[benben: reading-list | HackMD](https://hackmd.io/@benben6515/reading-list)

- 大先生をフォローする

転職前、たまにプログラミングの記事を読んでいました。筆者にとって最も关键だったのは、Huli 大先生の [程式自學十年心得（中国語）](https://buzzorange.com/techorange/2016/04/13/self-study-program-with-datastructure-and-algorithm/) です。これを見たのは 2018 年末頃でしょうか。筆者はまだ警備員で、独学でも 10 年かかるのか！やっぱり大変だな！えっ？この人誰？名前が変わってるね。胡立？ネット名？それとも胡瓜と関係あるの？

> 関連読書：[Huli 程式自學十年心得 | TechOrange](https://buzzorange.com/techorange/2016/04/13/self-study-program-with-datastructure-and-algorithm/)

筆者の仕事は日夜交代制でした。その日はたまたま夜勤で、退勤までまだ時間があり、空も明るくなっていませんでした（航空関連の仕事は十中八九、交代制です。飛行機は 24 時間飛んでいますよね？なら 24 時間誰かがいなければならない、その通りです）。この人のブログでも見てみよう。見ないほうがよかった、見てびっくり。この人は私と同じくらいの年齢で、大学も出ていないのに、もうシニアエンジニアだったのです！

胡立大先生には本当に感謝しています。彼は自分の物語をシェアしてくれました。筆者が今自分の物語をシェアしているように。この道が可行だと感じさせてくれました。だから筆者はその場で **一つの選択** をし、**努力して実行** しました。筆者は私立の応用数学系でした。数学はそれなりに得意だと思っています（學測数學は満点。なのに私立なのは聞かないで）。転職もなんとかなるだろうと、仕事しながらゆっくり独学し、とっくに昼も夜も区別がつかなくなっていたので、後からフルタイム学習プログラムに参加してさらに強化しようと思い、今に至ります。

仕事をするようになると、主に Vue フレームワークを使い、ドキュメントを読んでいるうちに Vue のコアメンバー —— **antfu** 大先生を見つけました。GitHub ランキング 100 位以内。これ以上は言いません。本当に強くて控えめ。多くのツールをオープンソース化し、記事やリソースも全部公開しています。あなたが使ったことがあるものもあるかもしれません。

2022 Vue X Laravel Conf では Vue セッションを聴きに行き、特に antfu にサインを頼みました。彼は「サインしたことない」と言っていましたが、サインしてくれました（照）。Yes！こうして **史上初の antfu のサイン** ゲット！大先生をアイドルのように追う感じですね！

> 詳しくは：[Laravel x Vue Conf Taiwan 2022](https://laravelconf.tw/)

とにかく、自分の分野の大先生（あるいは身近な人でも可）を何人かフォローすると、もっとモチベーションが出ます。**あなたの道しるべとなる灯り** のように。迷ったり諦めそうになった時も、その灯りは点き続けています。

- 小まとめ

では筆者は他のフロントエンドと何が違うのか？大きな違いはありません。仕事では同じように CSS を切り、Api を繋ぎ、ドキュメントを書いています。私はシニアでしょうか？そもそも **シニアの定義** は何でしょうか？私にも分かりません。でももう重要でもなさそうです。**進歩し続けられることが最も重要** だと筆者は思います。職称にこだわる必要がありますか？

## 幸せですか？

- 開発者は素晴らしい仕事ですか？

正直言って、特別な感情はありません。**開発は仕事ですし、警備も仕事です。**仕事に貴賤はないと思います。**真面目にやる警備員はいますか？**必ずいます。R 社の下の警備のお兄さんはとてもいい人で、誰にでも挨拶してくれました。筆者も時々飲み物をご馳走していました。**サボる開発者はいますか？**私に言わなくてもいいです。多くの人が痛いほど分かっているはず。

重要なのは、あなたがこの仕事をどうやるかです。

以前、ある航空警備（aka 警備員）で働いていた時も、いろんな人の顔を見ました。横柄な機長がいて、会社に着いた時、駐車証を取るか取らないかで、まるで会社が私的な駐車場かのよう。とても礼儀正しい飛行教官（機長を教える立場で、機長を物理的に「飛ばせる」ほどの人）もいて、よく気にかけてくれました。「寒いから厚着してね」とか、飲み物を奢ってくれたり。面接に来た客室乗務員候補たちはみんなそれっぽく見えますが、書類確認の時に取るか取らないかの方もいました（実は私たち警備は協理や副社長と仲が良く、ある副社長は「言うこと聞かない奴は番号を書いて知らせろ、面接に来なくていいと言ってやる」と言っていました。実は私たちは隠れた面接の一次関門でした）。あ、話が逸れました、戻りましょう。筆者もこの仕事に感謝しています。もしかしたらこの仕事があったからこそ、筆者はあんなに必死にやれたのかもしれません。あるいは元から必死にやるのが習慣だったのかもしれません。筆者にも分かりません。

もう一つ補足すると、筆者は普段、自分のことを「エンジニア」とは呼びません。この言葉は乱用されているからです。「私もエンジニアなんだよね」というタイプの友人に会ったことはありませんか？話してみたらプログラミング言語が一つも書けないことが分かったり。軽視しているわけではありません（言うなら、Web フロントエンドこそ軽視チェーンの最下層かも QQ）。ただこのまま会話を続けるのが本当に気まずいのです。どうやったらバラさずにドヤ顔を維持できるかを考えないといけない。そんな経験ありませんか？会話も一つの芸術ですね。

- 今の仕事はどうですか？

今の仕事は本当にハードです。筆者は煽り運なのかもしれません。明知山有虎、偏往虎山行（山に虎があると知りつつ、あえて山に入る）。どれだけハードか見てやろうという、苦労を栄養にする感覚です。若いうちにたくさん練習する機会を。もしかすると将来会社が飛躍するかも（絶対に私の錯覚ですが、錯覚）。

完全リモートなので、兼業初日、技術を管轄するフルスタックの人が「まず xxx.xx.xx.xx を ping して、レイテンシを見て」と言いました。

一般の junior フロントエンドは？え？ping？何を ping するの？彼と拳で戦わないで、中路を切りに行こうとする？

幸い筆者は [鳥哥の Linux 私房菜（中国語）](https://linux.vbird.org/) を少し読んでいたのですぐにターミナルを開いてコマンドを打ち、「現在のレイテンシは約 160 ms です」と返しました。

> 関連読書：[鳥哥私房菜 - 第 5 章、Linux よく使うネットワークコマンド ping](https://linux.vbird.org/linux_server/centos6/0140networkcommand.php#ping)

フルスタックの人は淡々と「悪くない。じゃあそのまま開発していこう」と返しました。

そして火鍋を食べながら歌を歌いながら（映画『邪不圧正』（原題：譲子弾飛）のネタ）、そのままタスクを割り当てられ、即開発開始。12 月（2022/12）から旧正月（2023/02）までずっと残業続きで、1 日だいたい 12 時間以上（正職＋兼業）コーディングしていました。

```markdown
- K = 公司 K

2021/04 （MTR05） 開始全職學習
2021/10 （MTR05） 結業
2021/11 （K）正職 K 公司，轉職上工第一天

2022/12 （R、C）正職 R 公司、兼職 C 公司
2023/02 （C）正職 C 公司、離開 R 公司
2023/03 （<= 現在在這）
...
```

- で、あの年収百万はどこ？

来ましたよ。

現在 C 社のオファー年収は約 50,000 NZD（NZD と TWD のレートは約 19:1）。会社はニュージーランド登記なので NZD で支払われます。外資と思うかもしれませんが、台湾にないからといって外資とは限りません。社長とコアメンバーは中国大陸の人で、インドの同僚もいます。そんな会社です。

NZD を簡単に換算すると、年間約 95 万 TWD。まあまあです。作業量を考えれば本当にまあまあ。

ちょっと待って！これじゃ百万じゃない！正直者とか言って、嘘つき！

弾を少し飛ばせましょう。上のタイムラインをよく見てください。筆者は実は昨年（2022/12）から C 社で 2 ヶ月「兼業」していました。ここでの兼業とは **1 週間 20 時間**（フルタイムの半分）の労働時間で計算されるので、実質 **1 ヶ月分多い** 給与になります。

だから 95 万プラス約 7 万の兼業給で 102 万になります。ただし、1 年間ちゃんと働けるかは別問題ですが（えっ？）。もう一度言うまでもありません。**筆者は正直者です**（殴打）、百万なら百万なんです。

だいたいこんな感じです。この「何書いてるのか分からない」振り返りが皆さんのお役に立てば。またね〜
