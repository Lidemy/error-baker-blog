## 0 はじめに

やあ、また会えて嬉しいです！

今回は **Vim** についてお話しします。Vim を紹介しますが、退屈な Vim の歴史は含みません。

Pro のように Vim を使えば、マウスは不要です。

それは本当なの？と思うかもしれません。まあ、理にかなっていると言えます。

さあ、始めましょう！

## 1 精神

まず、Vim とは何か？Vim はエディタです、それは間違いありません。しかし単なるエディタではなく、**ほぼどこにでも存在します**。開発者であれば、ソースコードの管理に Git を使うでしょうし、リモートのマシンにログインするとデフォルトのエディタが Vim であることもあります。

多くの人は Vim の力に気付いていません。それは残念なことです。昔の筆者も同じだったので、心配しないでください。私が Vim の影響力に初めて気付いたのは、あるテック系 YouTube チャンネルでした。初めて見た時、Vim の力に刺されました。

また、Vim の精神は多くのツールやプラグインにもあります。例えば、筆者は VScode の Vim プラグインを使ってこの記事を書いています。Chrome の Vim プラグイン（Vimium）を使ってウェブを閲覧することもあります。Vim はどこにでもあります。

## 2 Vim から抜け出すキー

Vim から抜け出す方法を知っていれば、`:q` を押して `Enter` するだけ。これで OK です。この `:q` コマンドは Vim で最も有名です。この 2 つのキーにはたくさんのミームがあります。

さらに、保存して終了したい場合は、`:qw` を入力して `Enter` します。あるいは、`:x` を入力して `Enter` するか、`ZZ` で保存して終了、`ZQ` で保存せずに終了できます。素晴らしいでしょう？これで好きな方法を使えます。Vim はとても面白いです。

Vim の最低限の知識は：

1. テキストを入力する方法
2. テキストを保存する方法

この 2 つのポイントは **あらゆるエディタ** の第一歩です。そもそも私たちがテキストファイルの編集をどう始めたか思い出してみましょう。ほとんどの人は、初めて `.txt` ファイルを編集する際に組み込みのテキストエディタを使ったはずです。

- 組み込みエディタでテキストを入力するには？

> 好きなテキストを入力するだけです。

- 組み込みエディタでテキストを保存するには？

> `Ctrl/Cmd + S` を押し、マウスで `X` ボタンを押して閉じるだけ。大したことありません。

ほら、同じ 2 つのポイントでしょう？ただ Vim では少し違うだけです。信じられないかもしれませんが、Vim は一般的な組み込みエディタより **より豊かで面白い** 機能を提供します。

## 3 Vim のモード

1. Normal mode（ノーマルモード）
2. Input mode（入力モード）
3. Select mode（選択モード）

初心者はまず **normal mode** と **input mode** を知っておけば十分です。

Vim の世界に初めて入ると、常に **normal mode** がデフォルトです（`.vimrc` ファイルなどを設定していない場合）。このモードでは、**特定のキーを押さない限り**、テキストを入力 **できません**。これは重要なので知っておきましょう。筆者もかつてネットでこれを学びました。でも「何も入力できない」ということは、意図的に特定のキーを押した可能性があるということに気付きました。そこで私は適当にキーを叩いてみました。結果はどうなったか？そう、私が作業中だったいくつかの設定ファイルを書き換えてしまいました。

私は normal mode を「ナビゲートモード」として捉える方が好きです。特定のキーでファイルや文書の中を見回すことができるからです。同時に、いくつかのキーは異なる機能を持つ他のモードに切り替えることもできます。

> 雑学：Vim には「イージーモード」もあります。`vim -y file.md` のようなコマンドでファイルを開くと、Nano エディタのようなイージーモードで開きます。つまり、Nano のようなシンプルなエディタを使いたい場合、Nano をインストールする必要すらありません。Vim のイージーモードを使えばいいのです。ただ、このモードは Vim の力を示せないので、おすすめしません。

## 4 方向

Vim の世界には 4 つの方向があります：

1. h (**←**)
2. j (**↓**)
3. k (**↑**)
4. l (**→**)

はい、ちょっと変ですよね！分かってます！でもキーボードを見てみてください。この 4 つのキーが右手の指の下に順に並んでいるのが分かります。偶然ではなく、マウスを使う機会を減らすことで理にかなっているのです。

従来の方向矢印キーの代わりに、Vim は **h、j、k、l** を使います。これにより、指をキーボードの中央上に保てます。それがどうしたと思うかもしれません。かつての筆者もそう思っていました。でも今は **はい、重要です** と言えます。筆者は 60% キーボードも使っています。これらは確かに作業への集中を高めてくれます。

Vim にはさらに効率的なナビゲーション方法もあります。例えば `H`、`L`、`M`、`ctrl + d`、`ctrl + r`。でも h、j、k、l にまだ不慣れなら、**とりあえずこの 4 つのキーに集中すれば十分で、Vim でやりたいナビゲーションはすべてできます**。これは本当です。違いは所要時間だけです。

## 5 入力方法

5 つの入力方法があります：

1. a、A
2. i、I
3. o、O
4. c、C
5. r、R

でもこれら 5 つのキーの大文字と小文字は少し違います。つまり、Vim には 10 通り以上のテキスト入力方法があるということです。

少し圧倒されるかもしれませんね、分かります。繰り返しますが、`i` だけ知っていれば十分です。なぜこんなにたくさんの入力方法があるのかと思うかもしれません。`i` と `a` を例に挙げると、両者はとても似ていますが、一方は空白箇所に **insert（i）** し、もう一方は空白箇所に **append（a）** します。

実は、Vim でテキストを入力するのに `i` と `a` を覚えているはずです。覚えていない？筆者の後に続いて：`i, insert`、`a, append`。大事なのは、コマンドを暗記するのではなく、声に出して言うことです。

## 振り返り

Vim に既に慣れている読者にとっては、この記事はとても簡単かもしれません。でも Vim は簡単です、1、2、3、4、5 のように。冗談です。

Vim は学ぶのが難しいです。でも一度学んでマスターすれば、Vim はあなたのスキルの中で最高の投資になります。それはコーディングだけでなく、文書編集（本当にあらゆる文書）にも当てはまります。

- 第一に、Vim の最低限の知識を学ぶ。

それは `h`、`j`、`k`、`l`、`i`、`:q!`、`:wq` です。これらだけで Vim のすべてができると信じましょう。

- 第二に、Vim をキーごとに学ぶ。

最低限の知識に慣れたら、次のキーに進み、キーごとに機能を確認していきましょう。おすすめは `w`、`e`、`b`、`zz`、`ctrl + d`、`ctrl + u`、`%` です。

- 第三に、ホットキーを暗記しようとしない。

暗記する代わりに、ドキュメントを見たり Google 検索する必要がなくなるまで、できるだけたくさん使いましょう。それは最初のホットキー `ctrl + c` と `ctrl + v` を学んだ時と同じで、Vim の世界にはもっとたくさんのホットキー（クールで魔法のようなもの）があります。そしてこれらのスキルはあなたと共に成長します。筆者はこの感覚が本当に好きです。RPG ゲームをプレイしていて、最初は大したことのない（下手をすればポンコツな）武器から始め、最後には伝説の武器になって主人にふさわしい存在になる、そんな風に。
