## はじめに

はいはい！筆者の第 5 期 Lidemy メンタープログラムが終わりました。みなさんお元気ですか？全体を通じて血と涙に満ちた、感動的な道のりでした ><。みなさんも無事卒業できたことと思います！

卒業後は少し楽になるかなと思っていたら、次は就職という大ボスが待っています。考えが甘すぎました。卒業後は記事を書く時間が増えるかなと思っていたのですが XD

この 6 ヶ月のプログラムは、長いようで短い。その間にいろんなことに取り組み、違うことを研究した人もいて、みなさんそれなりに成長したのではないでしょうか！この記事は、筆者が学習時代に「共同ノート読書会」を立ち上げたことの小さな振り返りです。以前、先輩方の会社に読書会があると聞いて、いいなと思いました。みなさんの会社に読書会があるかどうかは分かりませんが、あれば素晴らしいですし、なければ自分で始めてみましょう！

## きっかけ

なぜこの共同ノート読書会が生まれたのか。きっかけはこうです。筆者は Lidemy で学んでいた頃、いつもノートを取っていました。みなさんも筆者と同じく真面目に取っていたことと思います（笑）。ある日みんなで雑談しているとき（gather.town の上で。google meet の少し自由なバージョンのようなもので、自由に別のスペースに移動して雑談したり、会議したりできます）、私が **JavaScript30** という無料の学習リソースを勧めました。でも実は私自身も 5 章くらいしか見ていなかったのですが、本当に素晴らしいリソースだと思ったので、ふと「読書会をやろうよ」と思い立ち、テーマは JavaScript30 にしました。ちょうど初心者向けの学習リソースでもあり、初心者の私たちにぴったりでした。何人かで持ち回りで共有し、ついでに自分の解説力も鍛え、さらに他の人に向けて共有する。聞く人は私たちの仲間だけじゃなくて、gather.town 上の他の人も聞けます。

> 関連学習：[JavaScript](https://javascript30.com/)

JavaScript30 はずっといい無料リソースだと思っていました。閲覧には登録が必要ですが、みんなにシェアしたとき、みんなもいいねと言ってくれました。以前登録したことある人もいましたが、私と同じく登録しただけで全然見ていない（笑）。なんたって無料だから。無料だと「あ！まあ後でいくらでも見る時間はあるし」と思って、結局ずっと放置してしまいます。そう、私がまさにそれ。でも他人に共有するとなる話が違います。

他人に共有するというモチベーションがあると、必ず見て、理解しようと追い込まれます。少なくとも自分が共有する部分は理解していないといけません。だからモチベーションもだいぶ違います。「無料だからとりあえず貰っておいてホコリを被らせる」のと比べても、このモチベーションは圧倒的に強いです。一人で見るより理解度もはるかに深まります。これは筆者が参加してみての感想ですが、きっと同じように感じている人は多いはずです。

## ツールを選ぼう：HackMD / CodePen

まず、オンラインノートツールの **HackMD** を紹介しましょう。共同ノートなので「オンライン機能」は必須です。もちろん Notion などでも構いません。要はみんなが受け入れて、使う意欲のあるオンラインツールであれば OK。HackMD はオンラインの **markdown** エディタです。markdown とは、簡単に言うと github の `README.md` の `.md` ファイルのことで、素早くドキュメントの書式を書ける記法です。**HTML** の簡略版のようなもので、シンプルで分かりやすい反面、カスタマイズは少し難しいです。

次にオンライン IDE の **CodePen** です。コードを書く必要があり、かつリアルタイムに編集・実行できる環境が欲しかったので、軽量なオンラインプラットフォームはとても便利なツールだと思います。CodeSandbox など他のものでもいいかもしれませんが、私たちの JavaScript30 はネイティブの HTML、CSS、JavaScript しか使わないので、明らかに CodePen のほうが私たちのニーズに合っています。ということで、デモ用のプラットフォームとしてこれを選びました。

この 2 つのコアプラットフォームがあれば、共同ノートの準備はだいたい整いました。簡単な紹介を書けます。例えば `README.md` のようなファイルで、読書会の始め方やスケジュールなどを簡単に説明します。もっと簡単に言うと、「誰が、何を、いつ、どこで、何のために」をはっきり伝えること。そうしないとみんなが何をすべきか分からなくなります。

## さあ、共同作業を始めよう

ツールが揃ったら、作業開始です。もちろん自分の好きなツールを選んでも構いません。ただ、ツールはあくまで補助であり必須ではない、ということを忘れないでください。

筆者が立ち上げたので、まずは簡単なテンプレートを用意しました：

```markdown
# Title

## HTML

## CSS

## JavaScript

// 其他補充 ...

```

だいたいこんな感じです。全部ネイティブの HTML、CSS、JavaScript なので、この 3 つの側面から説明すれば間違いないです。メインは JavaScript ですが、HTML の構成や、たまに CSS の面白い仕掛けも見どころです。自分が学んだことなら何でも共有して OK です。

もし markdown 記法にまだ慣れていなくても大丈夫。**HackMD チュートリアル** を参考にしてみてください。10 分で使いこなせるとの触れ込みです。ただ筆者は、非エンジニアの友人にはまだ少し不親切かなと思っていて、非エンジニアの友人を勧誘するのはずっと成功していません XD

> 関連学習：[HackHD チュートリアル（中国語）](https://hackmd.io/c/tutorials-tw/%2Fs%2Fquick-start-tw)

また、HackMD にはスライドモード（side mode）もあります。ただ、内容の量をあらかじめ計算しておかないと、途中で切れてしまう問題があるので注意。共有時にはとても便利なので、ぜひ試してみてください。

CodePen のほうは、よりエンジニア向けです。HTML、CSS、JavaScript の基礎があって初めて、何をしているか分かりやすくなります。もし読書会の内容にそれらが不要なら、このツールにこだわる必要はありません。他のプラットフォームを使っても大丈夫です。

CodePen のインターフェイスはこんな感じです：

![CodePen interface](https://i.imgur.com/T0viLyL.png)

CodePen の使い勝手はこんな感じです：

![CodePen usage](https://i.imgur.com/D7lmpRR.png)

簡単に言うと 3 つの部分に分かれています：HTML、CSS、JavaScript。右側に現在の見た目がリアルタイムで表示されます。新しいアイデアがあれば試したり遊んだりでき、一種の PlayGround のような概念です。他の人に共有することもできます。

共同ノート読書会の立ち上げはこの辺で。ここからが一番難しい部分：「継続すること」です。

JavaScript30 全体を通じて学ぶことは多く、網羅的です。中には本当にあまり使わないものもありますが、間違いなく視野が広がります。「こんなのあったんだ！」と驚くこともあります。しかも全部ネイティブ。一通り見終わった後、「私、本当に JavaScript 分かってるの？」と疑いたくなりました。

JavaScript30 で面白かったもの：

- `console.log`、`console.error`、`console.count` など
- canvas での描画
- array の各種練習
- base64 ピクセル操作
- その他の面白いエフェクト

最後の簡単な成果物（参加してくれたクラスメイトに感謝）：[JS30 私は JavaScript マスターになりたい](https://hackmd.io/@benben6515/javascript-30)

## まとめ

今回も技術的な内容が少なめな記事でした（汗）。でも、これらのツールをよく知らない人はきっといるはずです。何かアイデアはあるけれどツールがなく、どう始めればいいか分からない——そんな人に向けて、この記事は書かれました。

実はこの ErrorBaker ブログ自体も共同ノートのブログです。だからあなたが望めば、一緒に共同ブログを立ち上げることもできます。あるいは、共同ノート読書会もいいなと思えば、それを立ち上げるのもありです。さあ、今すぐあなたの共同作業を計画し始めましょう！

最後に、みなさんと一緒に進歩し、成長していけたら嬉しいです。ご覧いただきありがとうございました。
