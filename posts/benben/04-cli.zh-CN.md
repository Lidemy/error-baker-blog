## 师父领进门

一开始，会接触 CLI（Command-Line interface）应该是在 Lidemy 中，最初的 week1。当中老师简单地介绍了 CLI 的操作——原来只用键盘就可以做这么多事！我对此深深著迷，因为离 **心中想像中的黑客** 又更接近了。你说有点中二（？）我敢说你小时候一定也想象过。但有时候就是这种微不足道的小动力反而一直推着你前进。

但是没有师父领进门，你也不会自己去研究这些东西。没有国文老师还会自己去学数学、没有数学老师还会自己去学公民，隔行如隔山啊。如果你想惹毛学校的老师，照我上面的去问看看，一定会被老师打。

我很幸运，刚好也喜欢 CLI 操作又有师父带领，所以稍微理解 CLI 的一些指令。下面也会介绍一下我认识的 CLI。当然如果有懂比较多的朋友，也可以交流一下，我会很开心的：D。

一开始接触的就是下面这门课，其实看试看的就很够了。

## 先求能用就好

工作上其实用鼠标就可以做到很多事了，但有时候非 CLI 不可的时候，你就会觉得「书到用时方恨少」。例如：需要远端操作主机的时候（好吧，纯前端工程师可能真的不太会用到）。

总之先来看看基础的指令吧！

- cd
  切换文件夹路径
- ls
  查看当前路径文件夹有什么文件或文件夹
- pwd
  查看当前路径位置
- touch（Windows 可能不适用，可以考虑先使用 git bash）
  新增文件

这几个指令会非常非常常用到，所以一定要记下来。但依笔者的学习过程，我认为：先掌握基本的指令就好，然后再慢慢增加指令，是最有效率的学习法！

例如：一次教你一堆指令，不用一个礼拜，两天后你就全忘光了。那不如就先记几个常用的吧。我觉得这也是老师故意的安排，只带我们认识几个常用的，而且在第一周就交了，为的就是让我们提早熟悉 CLI，因为之后会大量用到，越早熟悉越好！

最最最常用的应该就是 `ls` 了。刚学习时，一进来文件夹就先 ls 看一下有什么内容，cd 后再 ls 一下，没事就 ls 一下，渐渐地就记下来了！

还有一个不错用的指令：`whoami`。在自己电脑上使用可能无感，但如果是公司的电脑、有很多使用者的话，这个指令可以查看当前的使用者是谁。简单说就是让你的电脑知道「他爸是谁」。一不开心就问电脑 `whoami`，它永远不会反抗你！

很多指令都是有意义的。看完上面的 `whoami`，是不是不用特别记，你下次就记得了，是吧？

## 修行在个人

再来之后会用到 CLI 的地方大概就是项目了吧，很多文件也都会写一点基本的用法。

例如：

```bash
# 開啟專案
npm run start

# 開發專案
npm run watch

...
```

其实这些都算 CLI 的指令，甚至各个工具、框架也都有自己的 CLI，如：`vue-cli`、`git-bash`、`docker-cli`、`mysql-cli`……等。

你说这么多都要学也太累了吧！好在除了特别的功能不一样，大多通用的功能指令都是一样的，如上面的：ls、cd、pwd、touch 等等。

另外这几个也是开发中非常好用的指令：

- mv
  移动文件
- copy
  复制文件
- rm
  删除文件

## 微进阶的 CLI

以上的基本的部分都熟悉了，就可以寻找自己喜欢的指令，然后慢慢扩充它。很像玩游戏练技能的感觉，哇今天又多学了一个指令，感觉像是多学了一个技能，这样才能一直往前，不会固定在这边。

再来我要分享几个好用（但还是因人而异啦 XD）指令，也是我边东看看西看看学来的。

- open/start（Mac/Windows）
  开启文件
- code .
  开启 VScode
- shutdown
  关机（shutdown -s -t 10）、登出（shutdown -l）
- exit
  关闭 terminal（你说我点个 X 不就好了，信不信我打完 exit 比你使用鼠标精准地点到那个小小 X 还快。不信？好吧，我以前也不信，但学起来你就会信了）

另外下面这两个都可以拆好多好多好多好多文章讲了，要深可以深到马里亚纳海沟了……

- git
- vim

但笔者也非常喜欢这两个指令呢！它们底下分别又多了很多的指令。目前也是使用 VScode 搭配 vim 插件在开发。水真的深，但使用熟悉了，长期下来开发效率会提升很多的（而且还很潮？）

如果也有兴趣的朋友也可以 google 一下这两个关键字，应该也是很多资源，只是都很催眠就是了。例如：git 只会 pull/push 的话真的很可惜，还有很多强大的功能；vim 的话，可能只有会用的人才知道它的好。

> 延伸学习：[Git - document](https://git-scm.com/doc)
> 延伸学习：[Vim - document](https://www.vim.org/docs.php)

## 总结

以上稍微简单地介绍了一些 CLI。当然这水还很深，等著你我去探索。虽然我也还只是个小小的 junior，但我认为熟悉 CLI 是前往 senior 的门票之一。除了效率之外，CLI 也是跨入后端的必备技能，所以当然不能只停留在这里！

但这边多指令又该先学哪个呢？我觉得开心就好！

为什么要学这个？因为我开心、我喜欢这个。比起盲目地去学一堆，我认为无压力地去学一个 `whoami` 指令开心多了，你说是吧？
