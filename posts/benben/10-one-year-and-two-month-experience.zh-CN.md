## 我怎么敢

- 不满三个月就跳槽？

笔者第一间上班的公司是某数字设计公司 K。那时因应「元宇宙」风潮而跟上的，想说未来蛮有发展的，进去后才发现，我错了！主管/PM 根本不懂技术、常态加班等。不过也不能怪谁，反而学到了一课。看到关键字，如：元宇宙、虚拟货币、NFT 等等，都要小心，很可能只是拿某个新名词蹭流量，要好好研究公司背景。

同时也边做边观察。**试用期不是只公司在看你，你也在观察公司**。真的觉得不行就赶快换一间吧，不要浪费时间。也不要觉得没做满一年履历不好看什么的。举个例子：一个在同一间公司待五年的，是好的履历吗？还是只是能力不足跳不走，所以只能待着？你不知道，只有跟他共事才知道。年资不满一年就换真的没什么，想想你有多少个一年可以这样试？

后来到了还不错的小公司 R。因为想离开 K 公司，所以就打开了 104。那时候也是过年前吧，也有很多面试邀约。因为有了前一次面试经验，所以看起来很雷的公司就先筛掉一波了。跟 R 公司的 CTO 聊蛮多的，一聊就是 2 个小时，也是马上就拿到 Offer。Offer 也还不错，也没什么好犹豫的就过去了。

- 半年又请特休去面试？

笔者个人也是 **半年左右会整理一下自己履历**的人。也不用多说，应该很多大大都这样说过。定期去看看机会的同时，也看看自己还欠缺什么，或是自己的价值在什么水平。这都是对你有帮助的，何乐而不为？

担心别人眼光？免了吧。真的有多少离职后还会联络的朋友，何必被一个眼光耽误了你的未来呢？但是也不要「太夸张」刚进去就一直想换工作，一直看工作机会、一直去面试，也就是俗称的「换换病」，反而不充实自己，长期下来，只是变得「很会面试」罢了。笔者建议 3～6 个月左右的频率就好。当然你已经在很顶尖的公司或是高阶主管就不一定了。

- 小总结

其实笔者也算是蛮幸运的，后来 R 公司的 CTO 真的是个好主管，技术很深很广，写 App 兼 PM，不会乱开需求、不会乱开没意义的会，下班后完全不会吵你，群组安静得跟没人一样。对下面的人也是蛮好的，但是他也很忙就是了。小公司嘛！正常。

也不是说什么我要走了就不做事。我到离职前一天都还上好上满 8 小时，写文件，整理之前的 code。上班期间约一年，没有迟到早退过，都是全心全意地在为公司写代码，任务完成了，还想着有哪些地方可以优化的。可以说是对这份工作问心无愧。我觉得这样就够了，再继续做下去，我可能会更难抽离吧！

我会再离开 R 公司也是迟早的事。我的工作量几乎是另一个前端的 2～3 倍。再说一次 **笔者是个很老实的人**，当然 CTO 真的太忙了，可能真的没注意到。提完离职那天，下午跟 CTO 也聊了很多想法。他问我之后有没有想当主管，我马上说有！（不然我是为了什么一直充实自己？我想成为主管或资深，希望啦）。他只简单说了：是这样的话多做一点是好事，他的工作量换算下来应该可以抵 5 个人，我了不起就抵 2 个（打不了 10 个 QQ）。我不知道我在他心中是不是有达到他的要求，我也不晓得我的选择是不是正确的。但天晓得呢？

那天下班从内湖骑车回家上了麦帅一桥，夜空很清澈，旁边的 101 依然亮着微微的蓝光，慢慢地，闪着闪着。

## 我凭什么

- 作品集

很多前端开发者，非常讨厌整理作品集。我也是！

每当换一个平台，履历、作品集全都从新整理一次。名字、生日、地址，还有最麻烦的自传，想到就很累，所以我非常讨厌一直整理这些东西。在软件开发的领域中，古有云「Don't repeat yourself」，重复的事做一次就好了。而且一直写自己的资料、自传，某种程度也算 repeat yourself （笑）。

身为一个前端工程师，自己做个网站，应该算是稀松平常的事了。所以帮自己的作品集好好整理一下，做个列表呈现一下作品集，也没什么困难吧？但这时候「很有热情」的前端开发者又不见了。每个口口声声都说对前端很有热情，但 github 上怎么空空（但也不排除都是私人项目或是在其他平台啦）的呢？所以啊，跟笔者一样当个老实的人吧。没有的话，也宁愿不要说有，有热情的话自然就会呈现出来了，不需要特别装。

做完履历、作品集网站后，加上笔者有一些 **个人原创**、**合作**的作品集，认真要讲每一个都可以讲个一小时起吧。也呼吁大家 **请不要抄作品集**，这样的话，履历的回复率就会高很多。笔者把秘诀都交给你了，别说笔者私藏啦 XD。

- 演算法

很多前端开发者，非常讨厌刷题。我也是！

首先，我讨厌的是「刷题」，但我喜欢「解题」的感觉。主要用 JavaScript、TypeScript 刷，也是前端的主要语言，可以一起熟悉语言跟逻辑。现在语言跟语言的差异越来越小了（当然还是有分动态、静态；强型别、弱型别）。别人做得到的，JavaScript 也做得到（等等！没有要战语言），拿来刷题也没什么不可以，不用硬要跟别人一样用 Python、C++ 等等（真的要选的话选 C++ 吧）。

我觉得刷题练的不是各个程序语言，而是练逻辑思考的。其他底层实现只是一些细节，如：宣告一个变数要用 `let`、`const`、`int`、`bool`、`char*` 之类的，就用自己熟悉的语言即可，之后再来进行第二、第三语言的挑战。

今年 2023/01/15 去了 [2023 Yourator 数字职涯博览会](https://www.accupass.com/event/2211140558381956060137)。笔者到处问前端面试会考 leetcode 吗？得到的答案大多是 **不考或最多考个 easy**。真的不喜欢刷题就不要刷也 OK，但相对的，就要拿出对应的实作经验来证明你的实力就是了。

- Vim

很多前端开发者，非常讨厌使用 Vim。我也是！

笔者的讨厌是指刚学 Vim 的时候，连个退出（`:q`）都卡半天，移动只会一格一格移动（`hjkl`），输入只会 `i`，保存退出只会（`:wq`）。但基本上这样就可以完成 **所有的任务** 了，只是……花的时间长短。

直到看到各种大神的神操作，苦练了一番。笔者真的是从上面那样的状态开始使用 Vim，慢慢写代码开始的，后来终于能在 vim 中飞来飞去、使用各种脚本，操作算是行云流水了。用上古神器写前端就是潮到出水（X

学习 vim 看似没什么太大的帮助，举个例子：vim 中 `:q` 表示 quit 的意思，很多软件也遵守这个传统，如：`git log` 下输入 `q` 也是退出的意思。还有像是远端 server 进去预设就是使用 vim 来操作。这时候，你会感谢以前的你已经把 vim 学好了！

常在 twitter 或是社团上，有人战编辑器的时候，总会看到 **real pro use vim** 这句话，表示专业的开发者使用 vim 开发。目前前端应该也没什么人在使用 vim 开发（等等！没有要战编辑器）。好用的工具也越来越多，也可以做到 vim 做得到的事，所以要不要学又是见仁见智了。

总之，可以 **提高你的产能的**都是好东西，或是可以试试 Emacs 或是 IntelliJ 都是不错的选择，但这两个工具部分笔者就没有涉猎了。

- 小总结

有没有发现 **当别人讨厌什么，你就去做什么**，这将会是你的优势。当然你也要付出心力去学习。优秀是什么？就是 **使别人变得平庸的能力**。笔者同意。你可能会觉得笔者很「卷」，但这也是没办法。抱歉，世界就是这么运作的。你会的技能越稀有且有需求，就是越是高级人才。或是想一下，Junior 跟 Senior 的差别是什么？为什么年资差不多，Senior 是别人而不是你？

等等！你说你也讨厌切版？也没关系啊，那去写后端 SQL 如何？

有一个工程师笑话是这么说的：前端不会切版、后端不会 SQL。

当然也很多工具们把这些事「包好」了，举个不是很恰当的例子：Bootstrap 取代切版；ORM 取代 SQL。但是当你要改动 Bootstrap 时却改不动；写一个简单 CRUD 的 SQL 时写不出来。这时候，你就会明白了。不然真的以为工程师每天复制/贴上、喝喝咖啡就有钱领了吗？哪里来的幻梦泡泡，笔者先帮你戳破。还是如果真的有这种工作的话，请马上跟笔者说还有没有缺人（咦？）。

## 我做了什么

- 继续地追 **新技术**，补你所欠缺的 **旧技术**

软件这一个领域，唯一不变的就是一直在变，这应该也讲到烂掉了。尤其是前端，框架、工具变化真的太快了！如果不跟上就等著被淘汰。React 都出到 18 了，只会 jQuery 够吗？Vite 打包快得跟闪电一样、Vue 3 也很香，不学一下吗？

追新技术对开发者来说也是家常便饭了，那……旧技术呢？**计算器概论、资料结构、演算法**。有趣的是，这些都是万年不变的旧技术，反而没什么人要学。比较起来这应该是 CP 值最高的技能，结果大家都在热衷新技术，甚至在战新技术哪个好。有时间战的话，不如拿去提升自我。这些技能都会的话又何必战呢？

人家资工本科训练了 4 年才有的功力，又怎么会觉得 3～6 个月就可以追上？还有时间的话，请好好地补完这些必要的技术知识。转职的光环会有用完的一天的。未来要跳槽的你，会感谢以前的你的（好饶舌），眼光要放长远啊。

- 追踪大神

在转职前，就偶尔会看一些程式的文章。笔者觉得最关键的一篇文章是 Huli 大神的：[程式自学十年心得](https://buzzorange.com/techorange/2016/04/13/self-study-program-with-datastructure-and-algorithm/)。看到这一篇大约是在 2018 年尾左右吧，那时笔者还是个保全，想说原来自学也要花个十年啊！果然不容易啊！咦？这个人是谁啊？名字好特别喔？胡立？难道是网名？还是可能跟胡瓜有关系？

> 延伸阅读：[Huli 程式自学十年心得 | TechOrange 科技报橘](https://buzzorange.com/techorange/2016/04/13/self-study-program-with-datastructure-and-algorithm/)

笔者的工作需要日夜轮班，那天正好值晚班，反正离下班时间还长，天也还没亮（只要跟航空有关系的职业十之八九都要轮班，想一下飞机是不是 24 小时在飞的？是，那是不是 24 小时都要有人在？那就对了），去看一下这个人的 Blog 好了。不看还好，一看不得了，这个人跟我差不多年纪、大学没毕业，已经是资深工程师了！

真的很感谢胡立大大，他把他的故事分享出来了，正如笔者正在分享自己的故事，让我觉得这条路是可行的。所以笔者当下 **做了一个选择**，并且 **努力地执行**着。笔者大学念的是私立的应数数学系，自认数学还行（学测数学满级分，不要问我为什么这样还要读私立大学），转职也应该还算 OK，就一路边上班慢慢自学，反正我早就无法分辨白天或黑夜。直到后来参加全职学习，想说再去加强一下，就一路加强到现在了。

后来工作后，主要使用 Vue 框架，文件看着看着，也发现了 Vue 的核心成员 —— **antfu** 大神，Github 排行前 100，其他就不多说了，真的很强又低调，开源了很多工具，有的你可能就有用过，也分享了很多文章、资源，全都开源的。

2022 的 Vue X Laravel Conf 笔者也有去听 Vue 的议程。笔者特别去找 antfu 签名，他还说他以前没有签过名的，但还是帮我签了（羞）。Yes！就这样 **史上第一个 antfu 的签名** Get！大概就是这种追大神当追星的感觉吧！

> 了解更多：[Laravel x Vue Conf Taiwan 2022](https://laravelconf.tw/)

总之，找几个在自己领域的大神（或是身边的人也都可以）追踪一下，会更有动力的，就像有 **一盏明灯在为你指路**，当你迷惘时、想放弃时，它依然那样亮着。

- 小总结

所以我跟其他前端有什么不一样？也没什么太大的差别，上班一样切切版、接接 Api、写写文件。我算资深吗？到底 **资深的定义**是什么呢？我也不知道，但好像也不是这么重要了。笔者认为 **能够一直进步才是最重要的**，何必拘泥在一个职称呢？

## 我开心吗

- 当开发者是个很棒的工作吗？

老实说，没什么特别的感觉。**开发是份工作；保全也是份工作**。工作嘛，我觉得并无高下之分。**有没有很认真的保全？**一定有啊，像 R 公司楼下的警卫大哥人就很好，看到每个人都还会问好，笔者就时不时会请他饮料；那 **有没有很混的开发者？**不用跟我说，一定很多人非常有感觉。

重点是你怎么做这份工作？

以前笔者在某航空警备（aka 保全）上班时，也看过很多人嘴脸。有很跩的机长，进公司时停车证爱拿不拿的，好像公司是他私人停车场；也有很有礼貌的飞行教官（负责教机长的，是可以把机长干到飞起来，物理上飞起来那种），常常关心我们，天冷了要多穿一点，有时也会请饮料给我们喝；也有来面试的空服员，看起来都人模人样，但有的对核资料时爱拿不拿的（其实我们警备大多都跟协理、副总很好，也有副总跟我们说不听话的，编号直接登记给他们，直接叫他不用来面试了。其实我们就是隐藏的面试第一关啦）。好，这边扯远了，回来。笔者也感谢这份工作，也许是因为有这份工作，笔者才这么拼命的，也可能本来就习惯这么拼了，笔者也不知道。

这边再补充一个。笔者通常不会称自己是工程师，因为这个词已经被滥用了，应该多少遇过那种「我也是工程师欸」的那种朋友，结果聊一聊才发现他一个程序语言都不会写。也不是说鄙视的意思（真要说的话，Web 前端可能才是鄙视链的底层 QQ），只是再聊下去真的很尬好吗？还要想怎么聊才可以让你继续装逼又不戳破你。不晓得大家有没有这种经验呢？聊天说话也是门艺术呢。

- 现在的工作又是如何呢？

但现在的工作是真的很硬。笔者可能就很犯贱，明知山有虎，偏往虎山行，看到底会有多硬，吃苦当吃补的概念。趁还年轻多一点练习的机会，或许搞不好之后公司起飞也不一定（一定是我的错觉啦，错觉）。

因为是全远端，兼职上班第一天，主要管技术的全端就说：「你先 ping 一下 xxx.xx.xx.xx，看一下延迟是多少」。

一般的 junior 前端应该是想说？蛤？拼？拼什么？不要跟他拼拳，尝试切他中路？

好在笔者还有读过一点 [鸟哥的 Linux 私房菜](https://linux.vbird.org/)，马上就是打开 terminal 下指令，回复：「目前延迟约为 160 ms」

> 延伸阅读：[鸟哥私房菜 - 第五章、Linux 常用网路指令 ping](https://linux.vbird.org/linux_server/centos6/0140networkcommand.php#ping)

全端就淡淡地回：「还行，那就这样开发吧」

然后就吃著火锅唱着歌（电影：让子弹飞 的梗），直接分配任务，直接上工开发，就一路从 12 月（2022/12）到过年（2023/02）都在加班啦，一天差不多都 Coding 12 小时（正职加兼职）以上。

- 所以我说那个年薪百万呢？

这不是来了吗？

目前 C 公司的 Offer 薪资年薪约为：50000 纽币（纽币比台币约 19:1）。因为公司是挂在纽西兰的，所以发纽币。有人可能会想说，哇！外商欸！不要以为公司不在台湾就算外商，老板跟核心成员都是大陆人，也有印度的同事，就只是这样的一间公司。

简单换算一下纽币，差不多就是 95 万/年 台币了，也还行。对应的工作量来看的话，真的还行。

修旦几勒！这样也没有百万啊！还说自己老实人？你骗人！

让子弹飞一会儿。仔细看上页的时间轴，笔者其实从去年（2022/12）就开始在 C 公司「兼职」了 2 个月。这边的兼职是指 **一周花 20 小**（就是半个正职的概念）的工作时间计算，所以等于是 **多了一个月**的薪水。

所以 95 万再加上约 7 万的兼职薪水了就是 102 万了。只是不知道可不可以做满一年就是了（咦？），就不用再说了吧。**笔者这个人很老实的**（被打），有百万就有百万的啦。

差不多是酱啦。希望这篇不知道在写沙小的心得有帮助到大家，我们下次见～
