## はじめに

（仕事が忙しくて、原稿が遅れています）

みなさんこんにちは、時間が経つのは早いものです！もう 11 月末、あと 1 ヶ月で今年も終わりますね！

Lidemy 第 5 期の就職活動期間は 12/12 まで。いよいよ最後のピリオドが打たれます。もう仕事を始めている人も少なくないでしょう。

筆者の新しい仕事も始まったばかり。前身はデザイン会社で、主軸製品は 3D/VR。以前の案件は外注でしたが、最近自社のフロントエンドチームを立ち上げているので、まあ一種の革新（？）と言えるかもしれません。

なのでフロントエンド全体の開発プロセスはまだ決まっていません。マネージャーはフロントエンド技術にはそれほど詳しくない（何度かのやり取りで分かりました）ですが、本人もそれは自覚しています（元々 3D モデリングをやっていたので）。なので分からないことは同僚に聞いて OK。フロントエンドの同僚がとてもできる人で、今は分からないことをその人と相談しています。ここの人たちは皆さん良心的で、とても親切です。

もちろん大変なところもあります。フロントエンド部門が立ち上がったばかりなので資源が少なく、参考にできる古い案件もほとんどありません。どれも最近の案件で、フレームワークは react だったり vue だったり行ったり来たりで、中身はめちゃくちゃ。例えば styles の中に `.css`、`.scss`、`.sass` が同時に存在したり（普通こうは書かないですよね？）。でも今後は vue / pug / sass と babylon.js（WebGL の 3D フレームワーク）、バックエンドは php と Laravel フレームワークに確定したそうです。

受託開発の会社なので、早くキャッチアップすることが求められます。私には适应性期間として 1 週間だけ与えられ、その後すぐプロジェクトを投げられました。その 1 週間で会社のフロー、Vue、babylon.js を学ぶ必要がありました。事前に Vue を少し見ていたのは幸いでしたが、それでも自主的に残って少しでも多く学ぶようにしました。同僚もみんな残っていたので、ある種のみなし残業的なノリです XD

それっていいの？私はいいと思っています。ジュニアのうちにたくさん学べるなら学んだほうがいい。給料は気にしすぎないほうがいい（度を超さなければ）。解ける問題が多ければ多いほど、自分の価値も高くなります。1、2 年もすればスキルがついて、自然にスカウトが来るようになります。そのとき誰が損していたか分かります（X）。だから長い目で見ましょう XDD

というわけで、週末も会社のドキュメントを読んでいます。これから会社を揺るがしに行ってきます（？）

> 以下本題。

## 師匠が入り口まで連れてきてくれる

私が最初に CLI（Command-Line Interface）に触れたのは Lidemy の最初の week1 でした。その中で先生が CLI の操作をざっと紹介してくれて、キーボードだけでこんなにもたくさんのことができるのか！と私はすっかり魅了されました。**心の中で思い描いていたハッカー** に一歩近づいた気がしたからです。ちょっと中二病っぽい（？）と思いますか？でも子どもの頃、絶対にあなたも想像したことがあるはず。時にこういうささいなモチベーションこそが、ずっと私たちを前に押し進めてくれるのです。

でも師匠が入り口まで連れてきてくれなければ、自らこういうものを研究しようとは思わなかったでしょう。国語の先生がいなければ自ら数学を学ぶことはないし、数学の先生がいなければ自ら公民を学ぶこともない。分野が違えば山も違うと言いますが、学校の先生を怒らせたいなら、上記のような質問をしてみてください。きっと叩かれます。

私は運良く CLI 操作が好きで、しかも師匠がいたので、CLI のコマンドをいくつか理解できるようになりました。以下では私の知る CLI を紹介します。もちろん、もっと詳しい方がいればぜひ交流してください。とても喜びます：D

最初に触れたのが次の講座です。試聴だけでも十分足ります。

## まずは動けばよし

仕事ではマウスだけでだいたいのことができます。でも時に CLI でなければならない瞬間が来て、「書到用時方恨少（必要になって初めて勉強不足を痛感する）」と思う羽目に。例えば、ホストを遠隔操作するとき（まあ、純粋なフロントエンドエンジニアには本当にあまり縁がないかもしれませんが）。

とりあえず基本的なコマンドを見てみましょう！

- cd
  フォルダのパスを切り替える
- ls
  現在のパスのフォルダにどんなファイルやフォルダがあるかを見る
- pwd
  現在のパスの位置を見る
- touch（Windows では動かないかも。まず git bash を使うことをお勧め）
  新しいファイルを作成する

これらのコマンドは非常によく使うので、必ず覚えないといけません。でも筆者の学習経験から言うと、まず基本のコマンドをマスターし、そこから少しずつ増やしていくのが一番効率的だと思います！

例えば、一度にたくさんのコマンドを教わっても、1 週間どころか 2 日もすれば全部忘れてしまいます。それならよく使うものをいくつか覚えるだけにしたほうがいい。これも先生の意図的な配慠だと思います。よく使うものだけを数個取り上げ、しかも第 1 週に教える。それは私たちに早く CLI に慣れさせるためです。後で大量に使うことになるので、早く慣れるほどいいからです！

最もよく使うのはおそらく `ls` です。学習当初、フォルダに入ったらすぐに `ls` で中身を確認し、`cd` したらまた `ls`、特にすることがなくても `ls`、というのを繰り返しているうちに、自然と覚えていきました！

もう一つ便利なコマンドは `whoami`。自分のパソコンだと実感が湧きませんが、会社のパソコンのようにユーザーがたくさんいる環境では、現在のユーザーが誰かを確認できます。簡単に言うと、自分のパソコンに「親は誰か」を教えるコマンド。機嫌が悪くなったら `whoami` と聞いてみましょう。絶対に反抗してきません！

コマンドの多くには意味があります。上の `whoami` を見た後なら、わざわざ暗記しなくても次には覚えているでしょう？

## 修行は自分次第

その次に CLI を使う場面といえば、たぶんプロジェクトでしょう。多くのドキュメントにも基本的な使い方が少し書かれています。

例えば：

```bash
# 開啟專案
npm run start

# 開發專案
npm run watch

...
```

これらはすべて CLI のコマンドと言えます。各ツールやフレームワークにも独自の CLI があります。例えば `vue-cli`、`git-bash`、`docker-cli`、`mysql-cli` など。

こんなにたくさん学ぶなんて大変すぎるって？幸い、特別な機能は違えど、ほとんどの共通機能のコマンドは同じです。上に挙げた ls、cd、pwd、touch などがそうです。

開発中にとても役立つコマンドをもういくつか：

- mv
  ファイルを移動
- copy
  ファイルをコピー
- rm
  ファイルを削除

## やや進歩的な CLI

上の基本に慣れてきたら、自分の好きなコマンドを探し、少しずつ広げていきましょう。まるでゲームでスキルを育てるような感覚です。「今日はまた一つコマンドを覚えた、スキルが一つ増えた気分」と思えれば、その調子で前に進めます。ここで立ち止まることはありません。

次に、私があちこち見て回って学んだ、便利な（とはいえ人によりますが XD）コマンドをいくつか紹介します。

- open/start（Mac/Windows）
  ファイルを開く
- code .
  VS Code を開く
- shutdown
  シャットダウン（shutdown -s -t 10）、ログアウト（shutdown -l）
- exit
  ターミナルを閉じる（X をクリックすればいいじゃん、って？信じるかどうかは別として、私が `exit` と打つほうが、あなたがマウスであの小さな X を正確にクリックするより速いんです。信じられない？まあ、私も前は信じてなかったんですが、覚えると信じるようになりますよ）

そして次の 2 つは、どちらも記事を大量に書けるテーマです。深掘りすればマリアナ海溝くらい深くまで行けます……

- git
- vim

筆者はこの 2 つのコマンドも大好きです！それぞれさらに多くのサブコマンドがあります。現在は VS Code に vim プラグインを入れて開発しています。本当に奥が深いですが、使いこなせるようになれば、長期的な開発効率は大幅に上がります（しかもなんかかっこいい？）。

興味がある方は、この 2 つのキーワードでググってみてください。リソースもたくさんあるはずです。ただ、どちらもかなり眠くなる内容ですが（笑）。例えば git は pull/push しか知らないともったいない——もっと強力な機能がたくさんあります。vim は、使える人にしかその良さが分からないかもしれません。

> 関連学習：[Git - document](https://git-scm.com/doc)
> 関連学習：[Vim - document](https://www.vim.org/docs.php)

## まとめ

以上、CLI についてざっくりと紹介しました。もちろんまだまだ奥が深く、あなたと私がこれから探検していくのを待っています。私はまだ小さなジュニアですが、CLI に慣れ親しむことは、シニアへのチケットの 1 枚だと考えています。効率以外にも、CLI はバックエンドに足を踏み入れるための必須スキルでもあります。だからもちろん、ここで止まるわけにはいきません！

でもコマンドが多すぎる中、どれを先に学べばいいの？私は「楽しいならそれでいい！」と思います。

なぜこれを学ぶのか？私は楽しいから、好きだからです。やみくもにたくさん学ぶより、ストレスなく `whoami` 一つを学ぶほうがずっと楽しい。そう思いませんか？
