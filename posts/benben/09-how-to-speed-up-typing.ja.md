## はじめに

みなさんこんにちは、ついに帰ってきました。最近ちょっと忙しかったのですが、今回お話しするのは「タイピング効率」です！

「タイピングが速い」は間違いなく投資する価値のあるスキルです。特にキーボードで長時間入力する人、例えば開発者、ライター、さらには **速記者** などにとっては。もしタイピング速度がものすごく速いなら、~~肝を潰す~~ エンジニアなんてやめて、速記者になったほうがいいですよ（興味のある読者は速記者の給料を調べてみて）。筆者は学習の過程でオンライン講座やライブ配信をたくさん見てきましたが、多くの優れた達人や神様たちがタイピングがとても速いことに気付きました。手が小さい私は、見るたびに羨ましくなりました。自分のタイピングもあんなに速かったらなぁ！と速いコーディングを妄想し始め、以下 3000 字省略……

先に断っておきますが、私がここで言っているのは **効率** であって、速度そのものではありません。効率も主観的な感覚です。なので、速度を上げるためのタイピング練習サイトを紹介するだけでなく、**「タイピングが速い感じ」** を増す他の方法も紹介します。興味があれば読み進めてください！

最高の IDE（統合開発環境）は VS Code だという人もいれば、Notepad++、VIM、Office Word だという人もいます。元 Facebook のエンジニアがなぜ Office Word なのかを語ってくれています。

最高の IDE、Microsoft Word を試してみよう！ちょっと待って！私は VS Code の癖で `! + tab`、`.wrapper > .item * 3`（Emmet 記法）を打って、すぐに `Tab` を押したのに、何も出ない！HTML のテンプレートがすぐに生成されるはずなのに。もうコード書けない。

表面は面白いですが、よく考えると恐ろしい。そう、私たちは現代の IDE に甘やかされているのです。

筆者的には、Word は大きい、いや違う！Word は高い。正規版の Word を買うのも安くありません。最高の IDE を使うのにも代償がいるようです！

それとも……まずは真っ白な txt を試してみましょうか！

最初の出発点に戻りましょう。メモ帳だけを開いて、ググらずに何かを自分で書けるか試してみてください。例えば一番簡単な Counter や TodoList など。

## 剣士編

- スキル：**両手キーボードの習熟度**

タイピング速度を上げる最も基本的な方法。何もなく、ただ練習あるのみです！

タイピング練習サイトはたくさんあります。一つひとつは紹介しませんが、いいなと思うサイトをいくつかシェアします。友達に教えてもらったものもあります。全部隠さず公開します。

おすすめの英文タイピングサイト：

- <https://www.ratatype.com/>
- <https://www.typing.com>
- <https://10fastfingers.com/typing-test/english>
- <https://qwerty.kaiyi.cool/>

小さなアドバイスですが、正しい指の運びを学びながら練習するのがいいです。短期的にはタイピングが遅くなるかもしれませんが、慣れてくると、昔の指運びでつまずいていた壁をずっと早く突破できることに気付くはずです。

- スキル：**自動スペル防御**

次は自動防御の部分です。タイピングが速いと、間違いも増えます。そんなときは VS Code のプラグイン [Code Spell Check](https://marketplace.visualstudio.com/items?itemName=streetsidesoftware.code-spell-checker) を試してみましょう。

これもよく語られるプラグインですが、本当に実用的です。特にタイピングが速いときは。function を書くときに名前のスペルを間違えると、コードベース全体の命名が全部間違ってしまうことがあります。後で直そうとすると大変なことになります。個人プロジェクトならまだしも、自分で自分を雷に撃たれ、踏み続けて慣れるしかありません。

でも協力プロジェクトだと冗談では済みません。例えば開発スケジュールが逼迫している中で、ある typo が生まれ、それがずっと使い回され、環境変数にもバックエンドにもデータベースにも使い回される。これは「自分で直せばいい」レベルの問題ではなくなります。なので絶対に typo を避けましょう。ましてや typo で他人を地雷に踏ませるなんてしないでくださいね。自動防御をオンにするだけです！

- スキル：**`Tab` 連打をしない**

次は **`Tab`（auto complete）を使わない**、そしてターミナルで `npm run dev` を探すためにひたすら「上」を押さない、ということ。これも多くの新人開発者がつまずく地雷です。まあ今の IDE は強力なので、最後まで `Tab` で繋げて「`Tab` エンジニア」になることができます。開発スピードは確かに速いですが、その代償として、タイピングが速くならない、さらによく使う API（例えば `document.getElementById('#app')`）すら覚えられない、ということが起きます。試しに `Tab` なしで完成できるか試してみてください。できるなら何秒かかりましたか？

読者の中には「そこまでする必要ある？`Tab` って便利じゃない？」と思う人もいるかもしれません。筆者も最初はそう思っていました。しかしある授業で、クラスメイトが [胡立](https://blog.huli.tw/about/) 大先生に「英語のタイピングを先生と同じくらい速くするにはどうしたらいいですか？」と質問しました。その答えの一つが「Auto complete を使わないこと」でした。その瞬間から、私はほとんど Auto complete を使わなくなり、とても役立っていると感じています。だからここでもこの考え方をシェアします。

念のため言っておくと、筆者は `Tab` の利点にも同意していますし、多くのシニア開発者も `Tab` を駆使しています。ただし彼らはすでにほとんどの API を指先で操り、タイピングも非常に速いことが前提です。彼らにとって `Tab` は純粋に時間の節約です。しかし私が見る限り、多くの新人開発者の `Tab` の使い方は違います。確かに時間は節約できていますが、考える機会も、タイピングを練習する機会も節約してしまっています。

> `Tab` を使うのは、各 `Tab` の意味を本当に理解しているときだけにしましょう。

## 魔術師編

- スキル：**隕石術**

ハードコアな部分を語った後は、そろそろ「魔法」を。VS Code プラグイン [Power Mode](https://marketplace.visualstudio.com/items?itemName=hoovercj.vscode-power-mode) を試してみましょう。

インストールすると、すぐに「隕石術」を習得できます。演出は本当に派手です。公式サイトのデモを直接見てください。ここでは再掲しません。

これを使うとコーディングはいつも 100 点、隕石が降ってきてもへっちゃらです！いや……社長、もう隕石を降らせないで、もう限界です。

ちょっとだけ使うと、隕石術は素晴らしいと思いますが、地震術の効果（副作用？）はなしにできませんか？

できます。設定のところで：

VS Code の設定 -> 「powermode」と入力 -> 下にスクロールして「Shake:Enabled」を見つける -> チェックを外す

これで完了です！

> あとは隕石連発するだけ（開発の方の隕石じゃないよ）！タイピングは 500 倍、1000 倍の爽快感！

## 盗賊編

- スキル：**残像**

盗賊編にやってきました。ここでちょっとサボって、パッシブスキル「残像」を手に入れましょう。

ここでは何もインストールする必要はありません。これはネイティブの VS Code の「隠し設定」だからです。デフォルトはオフで、意外と知らない人が多いようです。筆者もある大先生のライブ配信で発見しました。友人たちにシェアしたところ、みんな絶賛してくれました。

では筆者が「残像」の覚え方を教えましょう。

- VS Code の設定 `Cursor Smooth Create Animation` を使う
  1. VS Code の設定画面を開く（`cmd` + `,` / `ctrl` + `,`）。
  2. 検索：`smooth`。
  3. `Editor: Cursor Smooth Create Animation` を見つけてチェックを入れる。

## 番外編：商人編

- スキル：**買う！買わない時なんてない！**

ここは装備編です。優秀なエンジニア（？）として、自分の好きなキーボードを見つけ、好きな軸と組み合わせ、好きな打鍵感を選ぶ必要があります。でもこの部分は本当に人によります。簡単に言うと、とにかくたくさん試すこと。そうすれば自分の好きな軸、素材、打鍵感が分かります。もう一つ重要なのはニーズです。携帯性が必要か、RGB ライトエフェクトが必要かなど。

筆者も簡単に自分の体験をシェアします。手をキーボードの中央に保って効率を維持したいので、87%、60% キーボードを検討しました。テンキーは置いてあっても場所をとるだけなので、もっとスペースを節約できます。軸の部分も本当に人によります。最初は各軸の違いに詳しくなかったので、最初の一本は万能の茶軸にしました。最初はまあまあ良かったのですが、使い続けると右手の小指が少し痛むことに気付きました（正統派の指運びでコードを打つと、多くのキーが右手の小指で打たれることに気付きます）。その後、赤軸に変えました。より軽くて静かです。

市場でよく見る軸を簡単に言うと：

- 力：赤軸 < 茶軸 < 青軸
- 音量：赤軸 < 茶軸 < 青軸

でも各社の製品によって多少異なります。入門したい読者向けの簡単な紹介です（~~でもお金が問題でないなら、HHKB を試してみても~~）。キーキャップの素材部分ですが、私はいわゆる黄油手（手汗が多い）。素材は PBT かそれ以上のものを試すのがいいでしょう。ここも多くの人が紹介しているので、商人の皆さんはちょっと勉強してください！

## まとめ：超初心者編

- スキル：**全部欲しい**

スキルポイントには限りがあり、フロントエンドの学びの海には果てがありません。初心者たち、戻りなさい！

でも私は全部欲しい！なので、上記の方法を個人のニーズに合わせて全部一緒に試してみて構いません。あるいは何もせず、自分が気持ちいいキーボードを買うだけでもいいですし、会社のポンコツキーボードで磨くだけでもいいです。自分が「タイピング効率」を上げられたと思えればそれで OK です。

タイピング効率というスキルは、使い手と一緒に成長するもので、投資対効果も高いです！このソフトスキルを過小評価しないでください。タイピング速度を開発者に求める会社もあります。ある意味で、このスキルは新人開発者と中・上級開発者を区切るものでもあります。少し時間を投じる価値は十分にあります！

また、本記事で触れたスキルは、ある古典的なゲームの中身に似ている気がします。完全に年齢がバレてます（QQ）。読者の皆さん、認めないでね、思い出ですね。

最後に、みなさんが理想のハッカーに一歩でも近づけることを願っています。でも倫理的ハッカーでいてくださいね！
